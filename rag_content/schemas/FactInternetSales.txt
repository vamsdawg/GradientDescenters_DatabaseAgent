TABLE: dbo.FactInternetSales
======================================================================

DESCRIPTION:
Online/internet sales transactions (B2C). Each row represents one line item from a customer order. Primary fact table for direct-to-consumer sales analysis.

TYPE: Fact Table

======================================================================
COLUMNS:
======================================================================

ProductKey
  Type: int NOT NULL [FK → dbo.DimProduct.ProductKey]

OrderDateKey
  Type: int NOT NULL [FK → dbo.DimDate.DateKey]

DueDateKey
  Type: int NOT NULL [FK → dbo.DimDate.DateKey]

ShipDateKey
  Type: int NOT NULL [FK → dbo.DimDate.DateKey]

CustomerKey
  Type: int NOT NULL [FK → dbo.DimCustomer.CustomerKey]

PromotionKey
  Type: int NOT NULL [FK → dbo.DimPromotion.PromotionKey]

CurrencyKey
  Type: int NOT NULL [FK → dbo.DimCurrency.CurrencyKey]

SalesTerritoryKey
  Type: int NOT NULL [FK → dbo.DimSalesTerritory.SalesTerritoryKey]

SalesOrderNumber
  Type: nvarchar(20.0) NOT NULL [PRIMARY KEY]

SalesOrderLineNumber
  Type: tinyint NOT NULL [PRIMARY KEY]

RevisionNumber

UnitPrice

  Type: float NOT NULL

TaxAmt
  Type: money NOT NULL

  ShipDateKey → dbo.DimDate.DateKey

======================================================================
USAGE NOTES:
======================================================================

- Primary fact table for internet/online sales analysis
- Join to DimProduct, DimCustomer, DimDate for complete analysis
- OrderDateKey is primary date for sales (not ShipDateKey)
- SalesAmount includes tax and freight
- Use DISTINCT on SalesOrderNumber for order counting
