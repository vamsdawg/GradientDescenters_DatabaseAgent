TABLE: dbo.DimCustomer
======================================================================

DESCRIPTION:
Customer master dimension. Stores customer demographic and profile information. Links to DimGeography for location.

TYPE: Dimension Table

======================================================================
COLUMNS:
======================================================================

CustomerKey
  Type: int NOT NULL [PRIMARY KEY]

GeographyKey
  Type: int NULL [FK → dbo.DimGeography.GeographyKey]

CustomerAlternateKey
  Type: nvarchar(15.0) NOT NULL
  Sample: (numeric keys)

Title
  Type: nvarchar(8.0) NULL

FirstName
  Type: nvarchar(50.0) NULL

MiddleName
  Type: nvarchar(50.0) NULL

LastName
  Type: nvarchar(50.0) NULL

NameStyle
  Type: bit NULL

BirthDate
  Type: date NULL

MaritalStatus
  Type: nchar(1.0) NULL

Suffix

EmailAddress
  Type: nvarchar(50.0) NULL

YearlyIncome

TotalChildren
  Type: tinyint NULL

======================================================================
FOREIGN KEY RELATIONSHIPS:
======================================================================

  GeographyKey → dbo.DimGeography.GeographyKey

======================================================================
USAGE NOTES:
======================================================================

- NEVER include EmailAddress, Phone, AddressLine* (confidential)
- Join to DimGeography for location information
- YearlyIncome useful for customer segmentation
- Use FirstName + ' ' + LastName for full name
